# 项目文件结构说明（整理后）

本文档说明当前项目已按功能完成分类整理，便于后续交接、复现实验和维护。

## 一、顶层目录说明

- `01_raw_data/`：原始数据与原始需求文档
- `02_processed_data/`：数据处理后的标准数据（classic / fresh）
- `03_core_algorithm/`：核心算法代码与主 notebook
- `04_experiment_outputs/`：实验输出（落盘表格与图表）
- `05_scripts/`：可直接执行的处理/实验脚本
- `06_docs/`：说明文档（pipeline、交接、数据结构）
- `07_delivery_packages/`：可直接交付给组员的压缩包
- `99_archives/`：历史目录结构归档（避免误删）

---

## 二、各目录详细内容

### 1) 原始数据：`01_raw_data/`

- `cvrplib_sets/`：CVRPLib 原始实例集合
  - `A/`, `B/`, `E/`, `F/`, `M/`, `P/`
- `raw_snapshot/raw/`：处理前原始文件快照（`.vrp/.sol`）
- `source_requirements/CS5491 AI Project.docx`：课程项目原始要求文档

### 2) 处理后数据：`02_processed_data/`

- `classic/`
  - `base/`：classic 实例标准化 JSON
  - `meta/`：classic 元信息 JSON
  - `index.csv`：classic 全局索引
- `fresh/`
  - `fresh/`：fresh 扩展实例 JSON
  - `fresh_meta/`：fresh 扩展元信息 JSON

### 3) 核心算法：`03_core_algorithm/`

- `notebooks/benchmark_cvrp_clean (1).ipynb`：主实验 notebook
- `modules/`
  - `benchmark_experiment_workflow.py`：正式 benchmark + 消融实验编排
  - `benchmark_export_plot_utils.py`：落盘与图表导出工具
  - `__init__.py`

### 4) 实验输出：`04_experiment_outputs/`

- `formal_benchmark/`
  - `formal_benchmark_runs/`：每次 run 的明细表与图表
  - `ablation_seed_summary.csv`：按 seed 的消融结果
  - `ablation_aggregate_summary.csv`：消融汇总结果
  - `ablation_cost_bar.png`、`ablation_feasible_bar.png`
  - `formal_experiment_meta.json`
  - `smoke_test_export/`、`tmp_plot_test/`：调试验证输出

### 5) 可执行脚本：`05_scripts/`

- `process_cvrplib.py`：classic 数据处理
- `generate_fresh_dataset.py`：fresh 数据生成
- `run_formal_benchmark.py`：正式 benchmark 自动运行

### 6) 文档：`06_docs/`

- `pipeline_docs/`：
  - `classic_cvrp_pipeline.md`
  - `data_schema.md`
  - `qa_report.md`
  - `fresh_generation_protocol.md`
  - `fresh_data_schema.md`
  - `fresh_qa_report.md`
- `handover_notes/`：
  - `CVRP_LLM_Handover_Notes.md`
  - `落盘与图表化执行步骤.md`
- `delivery_notes/`：
  - `数据集交付说明.md`

### 7) 交付压缩包：`07_delivery_packages/`

- `classic_cvrp_dataset.zip`
- `fresh_cvrp_dataset.zip`

### 8) 历史归档：`99_archives/`

- `legacy_layout/`：整理前的旧目录（`new/`, `docs/`, `delivery/`, `scripts/`, `data/` 等）
